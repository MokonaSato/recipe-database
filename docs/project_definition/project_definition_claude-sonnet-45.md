### 1.4 プロジェクトの目的
多忙な社会人が自炊の頻度を上げられるよう、レシピ収集から献立提案、買い物リスト作成までを生成AIがサポートするアプリケーションを開発する。ユーザが献立を考える負担を大幅に軽減し、健康的な食生活の維持を支援する。

### 1.5 プロジェクトの背景
プロジェクトオーナー自身が仕事で多忙な生活を送っており、自炊をする気が起きないという課題を抱えている。食事をおろそかにすると肌荒れや風邪など体調面に影響が出るため、自炊の頻度を上げる必要がある。

自炊を妨げる最大の要因は「献立を作るのが億劫」であること。具体的には以下の点が負担となっている:
- 食べたいメニューを複数思い浮かべること
- 献立同士の相性や栄養バランスを考えること
- 同居人の好みを勘案すること

日頃から好きな献立のレシピを貯めておき、AIに相談すれば1週間の献立と買い物リストを提案してもらえれば、自炊のハードルを下げられる見込みがある。

このような困りごとを抱える社会人は日本に多数存在すると考えられ、同様の課題解決を広く提供できるサービスを目指す。

## 4. 要件概要

### 4.1 機能要件
1. **レシピ収集機能**
   - 概要: ネット記事やTwitterからレシピを収集・保存できる
   - 優先度: 高

2. **AI対話による献立提案機能**
   - 概要: 生成AIがユーザと相談しながら1週間分の献立を提案する
   - 優先度: 高

3. **買い物リスト作成機能**
   - 概要: 提案された献立から必要な食材をリスト化する
   - 優先度: 高

4. **レシピ探しの働きかけ機能**
   - 概要: ユーザが積極的にレシピを探さなくても、システム側から働きかけを行う
   - 優先度: 中

5. **同居人の好み考慮機能**
   - 概要: 献立提案時に同居人の好みを勘案する
   - 優先度: 中
