## 2. 背景・目的
### 2.1 背景
- 作成者本人が、仕事で多忙な生活を送っており、自炊を継続することが難しいという課題を抱えている
- 忙しさから食事が疎かになり、肌荒れや風邪など体調面への悪影響が発生している
- 自炊における最大の障壁は「献立を考えること」であり、以下が特に負担となっている
  - 食べたいメニューを複数考えること
  - 献立同士の相性や栄養バランスを考慮すること
  - 同居人の好みを勘案すること
- レシピは主にインターネット記事やTwitterで閲覧するが、自ら積極的に探すことは稀である
- レシピ収集や献立検討を支援する仕組みがあれば、自炊頻度を上げられる可能性があると考えている

### 2.2 目的
- 作成者本人の「献立を考える負担」を軽減し、自炊の頻度を上げること
- 生成AIとの対話を通じて、1週間分の献立と買い物リストを提案する仕組みを実現すること
- 同様の困りごとを抱える日本の社会人ユーザの課題も解決できるサービスを提供すること
- 優先度は以下の通りとする
  - 第一優先：作成者本人の困りごとの解消
  - 第二優先：作成者以外のユーザの困りごとの解消

---

## 3. プロジェクト概要
### 3.1 概要
- レシピを収集・蓄積し、生成AIがユーザと相談しながら1週間の献立を提案するアプリケーションを開発する
- 提案された献立に基づき、必要な食材のお買い物リストを生成・提示する
- ユーザは主にiPhoneを利用することを想定している
- レシピ閲覧元として、インターネット記事やTwitter由来のレシピが中心となる

### 3.2 スコープ
#### 対象範囲（In Scope）
- レシピをユーザが保存・蓄積できる仕組み
- 生成AIとユーザの対話を通じた1週間分の献立提案
- 献立に基づくお買い物リストの作成・提案
- ユーザに対するレシピ探しの働きかけ（受動的なユーザを想定）

#### 対象外（Out of Scope）
- 具体的な除外機能は現時点では定義されていない
