## 2. 背景と目的 (Why)

### 2.1 現状の課題

* 仕事が多忙で自炊の意欲が湧かず、食生活の乱れから肌荒れや風邪などの体調不良を引き起こしている。
* 自炊において「献立を考えるプロセス」が最大の障壁となっている（食べたいメニューの想起、相性・栄養バランスの考慮、同居人の好みの反映などが非常に億劫）。
* レシピ探しが受動的（ネット記事やSNSが中心）であり、自分で探す手間も負担。

### 2.2 プロジェクトの目的

* 生成AIとの対話を通じて、ユーザーの好みや状況に合わせた1週間の献立と買い物リストを自動生成し、自炊の開始ハードルを劇的に下げる。
* ユーザーの健康維持と、献立作成に伴う心理的・時間的コストの解消を実現する。

## 3. 目標と成功の定義 (Goal)

### 3.1 主要目標 (KPI/成果物)

* **定量的目標:** [未設定：例：週の自炊回数の増加数など]
* **定性的目標:** 献立作成によるストレスの解消、同居人も満足する食卓の実現、体調の改善。

### 3.2 成功の定義

* [ ] ユーザーが保存したレシピを元にAIが1週間分の献立を提案できること。

* [ ] iPhoneユーザーが使いやすいインターフェース（ブラウザまたはアプリ）で提供されること。

## 4. スコープ (Scope)

### 4.1 実施すること (In Scope)

* 好きなレシピ（Web記事、SNS等）をアプリ内に収集・保存する機能。
* 生成AIによる献立の提案（ユーザーとの対話、同居人の好み考慮を含む）。
* 献立に紐づいたお買い物リストの自動作成。
* ユーザーに対して受動的なレシピ探しを支援する働きかけ（通知・レコメンド等）。
* iPhoneでの利用を最適化したUI/UX。
